# PaperAgent 工科学位论文爬取测试集（100篇，题录链接版）

> 生成日期：2026-06-24  
> 用途：用于 PaperAgent / TopicPilot-CN 的“论文链接抓取 → 题录解析 → 实验需求判断 → 选题可行性评级 → 用户审核”闭环测试。

## 0. 口径说明

- 本文件不是“下载后的全文数据集”，而是从公开可访问的中国学位论文题录入口中整理出的 **100 条工科论文候选样本**。
- 每条样本保留原始题录/论文入口链接，便于 PaperAgent 二次爬取、解析、验证和人工审核。
- 没有打包论文 PDF、全文、图片或数据库受限内容；这更适合作为合规的测试集构建方式。
- “实验需求 / 难度 / 周期”是基于题名、公开题录摘要片段和常见工程实现经验做的一级标注，后续可以作为 PaperAgent 的预标注，再由用户在工作台中修正。

## 1. 为什么这样构建

PaperAgent 当前项目定位不是自动写论文，而是把题目、论文、数据集、GitHub 工程、PDF/截图/网页材料整理成可审核证据链。因此测试集更应该考察：

1. 能不能保留原始链接并做 URL verified。
2. 能不能从论文题录里抽出任务对象、方法词、实验平台、数据集需求。
3. 能不能判断这篇论文对应的项目是否“好水”“稳中求新”或“高风险”。
4. 能不能避免把只在摘要里出现的内容脑补成已验证事实。

## 2. 字段定义

| 字段 | 含义 | PaperAgent 应测试的能力 |
| --- | --- | --- |

| year | 学位年份 | 时效性判断 |
| domain | 工科方向归类 | 领域分类 / 检索增强 |

| difficulty | 项目难度 | 五档可行性判断 |

| repeatability | 毕业前可迭代次数 | 风险提示 |
| paperagent_test | 该样本主要测试点 | Agent 评测目标 |

## 4. 推荐测试集拆分

| 子集 | 建议数量 | 用途 |
| --- | ---: | --- |
| smoke_test | 20 | 每次提交后快速验证：URL 抽取、标题解析、基础可行性标签是否稳定。 |
| regression_test | 60 | 常规回归测试：覆盖 YOLO、缺陷检测、裂缝检测、SLAM、机器人、电力巡检。 |
| hard_case_test | 20 | 专测高风险：三维点云、机器人硬件、自动驾驶多模态、医学/人体、能源装备实验台。 |

推荐的抽样策略：不要只抽 YOLO 论文。YOLO 类容易让系统误以为所有题目都“单卡可跑”，所以 hard case 必须保留。

## 6. PaperAgent 测试任务设计

### 6.1 任务一：题录抓取与链接保真

输入：`source_url`。输出：`title / year / source / abstract_snippet / verified_status`。

建议指标：

| 指标 | 定义 | 合格线 |
| --- | --- | ---: |
| URL 保真率 | 输出报告中保留原始链接且未替换、未伪造的比例 | ≥ 0.98 |
| 题名抽取准确率 | 抽取题名与测试集题名完全或近似一致的比例 | ≥ 0.95 |
| 年份抽取准确率 | 抽取年份与测试集一致的比例 | ≥ 0.90 |
| 无全文降级正确率 | 无法访问全文时，是否降级为“题录/摘要级证据”而不是编造全文内容 | ≥ 0.95 |

### 6.2 任务二：实验需求标签抽取

输入：题名 + 题录摘要片段。输出：`compute_need / data_need / hardware_need / deployment_need / risk_tags`。

建议标签：

- `single_gpu_ok`：单卡 GPU 可完成主要训练。
- `cpu_or_light_gpu_ok`：传统算法、SCADA、可靠性分析等不强依赖 GPU。
- `large_gpu_optional`：点云补全、多模态融合、大规模 3D 网络建议大显存，但 H100 不是必要条件。
- `h100_level_not_recommended`：如果系统判断必须 H100，通常应给出“不适合毕业设计复现”的提示。
- `self_collected_dataset`：需要自采数据或企业/现场数据。
- `public_dataset_available`：公开数据集即可启动。
- `hardware_platform_required`：需要机械臂、机器人、相机、Jetson、结构光、LiDAR、ZED 等硬件。
- `annotation_heavy`：需要大量标注。
- `domain_data_permission_risk`：医疗、企业生产、电力巡检、SCADA 等数据权限风险。

建议指标：

| 指标 | 定义 | 合格线 |
| --- | --- | ---: |
| 实验需求 Macro-F1 | 对上述标签做多标签 F1 | ≥ 0.75 |
| 数据风险召回率 | 对 `self_collected_dataset / domain_data_permission_risk` 的召回 | ≥ 0.85 |
| 硬件风险召回率 | 对机器人、机械臂、相机、LiDAR、Jetson 等硬件需求的召回 | ≥ 0.85 |
| H100 误判率 | 把普通 YOLO/U-Net 项目误判为 H100 级需求的比例 | ≤ 0.05 |

### 6.3 任务三：项目难度与周期评估

输入：论文题名 + 摘要片段 + 可用数据源。输出：`difficulty / cycle / repeatability / graduation_feasibility`。

建议将难度分为四档：

| 难度 | 解释 | 适合的论文定位 |
| --- | --- | --- |
| 低-中 | YOLO/U-Net/传统检测，公开数据或小规模自采即可 | 保毕业、项目展示 |
| 中 | 有完整训练与消融，但不强依赖硬件 | 稳中求新 |
| 中-高 | 点云、SLAM、三维、双目、时序、多模态 | 稳中求新或冲高水平，但要砍范围 |
| 高 | 机械臂/机器人硬件、医学合规、复杂强化学习、多模态攻击防御 | 不建议作为短周期毕业主线，适合作为扩展方向 |

建议指标：

| 指标 | 定义 | 合格线 |
| --- | --- | ---: |
| 难度准确率 | 完全匹配一级标注 | ≥ 0.70 |
| 邻档准确率 | 预测档位与标注最多相差一档 | ≥ 0.90 |
| 周期邻档准确率 | `0.5–2天 / 1–3天 / 3–10天 / 1–3周` 等周期邻档命中 | ≥ 0.85 |
| 高风险召回率 | 对“高”或“中-高”项目的召回 | ≥ 0.85 |

### 6.4 任务四：报告生成质量

要求 PaperAgent 输出时必须区分：

- 题录事实：题名、年份、来源链接。
- 模型推断：实验需求、难度、周期。
- 未验证信息：数据集名称、硬件型号、具体指标。
- 用户可操作建议：保留、降级、换题、补数据、找 GitHub baseline。

建议指标：

| 指标 | 定义 | 合格线 |
| --- | --- | ---: |
| 支撑句比例 | 报告中每个关键判断是否能回溯到题录/摘要/链接 | ≥ 0.85 |
| 幻觉率 | 编造数据集、指标、GPU型号、作者结论的比例 | ≤ 0.05 |
| 降级建议可用率 | 高风险论文是否给出具体降级方案 | ≥ 0.80 |
| 人工审核触发率 | 信息不足时是否进入 pending/human_review | ≥ 0.90 |

## 10. 注意事项

- 这些链接主要是题录入口，不代表全文一定开放。PaperAgent 遇到受限内容时应保留链接、输出题录级证据，并提醒用户手动上传 PDF。
- 对“实验需求”的判断不能只看模型名。比如 YOLO 可能单卡可跑，但如果题目涉及无人机巡检、机械臂抓取、结构光或双目系统，硬件和标定才是主风险。
- 对“实验周期”的判断应保守。能跑通训练不等于能完成论文级实验，还需要数据清洗、标注、消融、可视化和报告复核。
- H100 不是本测试集的默认需求。绝大多数硕士工程项目应用单卡 3060/3090/4090 或云 GPU 即可；真正需要警惕的是数据和硬件。
