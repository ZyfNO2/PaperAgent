## 4. 改造路线

### P0: 前端卡片化 (1-2 天)

**A. 论文卡片 (Phase 04)**
- `papers[]` 每条渲染 1 张卡片
- 显示: title / authors (前 3 + "+N") / year / source (badge) / abstract 折叠 / url (按钮)
- 来源 badge 颜色: arXiv=蓝, LLM-generated-candidate=灰, user-uploaded=绿
- "查看 PDF" 按钮 → 新窗口打开 pdf_url
- "删除" 按钮 (可选, 后续)

**B. 数据集卡片 (Phase 04)**
- name / scale / license / fit_to_topic (badge 颜色)
- "下载" 按钮 (如果 download 存在)

**C. Baseline 卡片 (Phase 04)**
- name / repository_url (按钮) / 复现难度 (badge 颜色: 低=绿 / 中=黄 / 高=红) / fit_to_student_resources

### P1: 上传文献 (1 天)

**前端**: Phase 04 卡片上方加 1 个『添加论文』按钮 → 弹出表单
- 必填: title
- 选填: authors (逗号分隔) / year / url / abstract
- 提交: POST /api/v1/projects/{id}/papers (新端点)
- 后端: 接受 paper, 追加到 EvidenceLedger.papers (source="user-uploaded"), 重新评级
- 论文卡片: user-uploaded 来源标绿色, 排在 arXiv 论文前面

**后端新端点**:
- `POST /api/v1/projects/{id}/papers` body: {title, authors, year, url, abstract} → upsert EvidenceLedger
- `GET /api/v1/projects/{id}/papers` → 列出当前所有论文 (不暴露 evidence_ledger_id 字段)
- 留 `DELETE` 后续

### P2: 评分信号展示 (半天)

**前端**: Phase 05 卡片下面加折叠区『评分细则』
- 每个 DimensionScore 展开看:
  - **加分项** (绿色 ++): "方向论文多 (5 篇) +20"
  - **减分项** (红色 --): "无综述 +0"
  - 总分 0-100 + 评级 A/B/C/D
- 折叠区不默认展开, 用户想看时点开

**后端**: DimensionScore.pluses / minuses 字段已加 (上次改动), 验证 flow 端点能透传

### P3: 修 undefined bug (1 小时)

**问题**: Phase 07 卡片 `proposal_sections=undefined / innovation_count=undefined`

**根因** (待验证):
- runPhase7a 写 `state.phases[7].data.proposal = r` (proposal 字段)
- runPhase7b 写 `state.phases[7].data.committee = r` (committee 字段)
- 卡片渲染只读 committee.rating, 没读 proposal_meta.section_count

**修法**: 卡片合并 proposal_meta + committee_meta, 读两个 source

### P4: Playwright e2e (半天)

- 改 e2e 用 v2 UI selector (stepper / dynamic phase-panel / trace 面板)
- 跑过 happy path + 论文卡片断言 + 上传论文断言
- 跑过 8 phase

### P5: 浏览器自测 (半天)

- 起 Playwright 浏览器, 用 sync_api 模拟用户点击
- 每步截图 `tmp/click_step_NN.png`
- 验证: 表单 → 流式 → 论文卡片 → 上传 → 评分信号 → 导出

## 5. 反直觉检查清单 (每次前端改动必过)

- [ ] 用户能一眼看出当前在哪个 phase? (stepper 进度 ✓)
- [ ] 主按钮文本是否说人话? (✓ 已完成 / 下一步 / 拼装中)
- [ ] 论文卡片能否快速扫到 title / 来源 / 摘要?
- [ ] 评分细则是否藏在折叠区 (默认隐藏, 想看时能看)?
- [ ] 上传论文后是否立刻看到新卡片?
- [ ] Phase 07 卡片是否同时显示 proposal (sections) + committee (verdict)?
- [ ] 错误信息是否给明确下一步? (如: "无数据集候选, 请先在 Phase 04 添加")
