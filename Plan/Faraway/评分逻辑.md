# TopicPilot-CN Phase 05 评分逻辑

> 文件：`packages/agents/nodes/phase5_risk.py`
> 输入：`EvidenceLedger` + `TopicSpec` + `ProjectIntake`
> 输出：`RiskScore.overall_rating (A/B/C/D)` + 6 个 `DimensionScore` (各含 pluses/minuses)

---

## 1. 总体框架

```
6 个维度评分 (0-100)  →  平均分  →  按 goal_level 阈值  →  A/B/C/D
                       ↓
                  pluses / minuses (前端展示给用户)
```

**6 个维度**：
1. 方向成熟度（论文 + 综述 + 学位论文模板）
2. 数据可得性（datasets + 继承资源）
3. baseline 清晰度（数量 + 复现难度）
4. 实验可行性（metrics + experiment_templates）
5. 工作量可拆性（WP 数量 + wp_binding 覆盖）
6. 毕业时间风险（首张结果表 + 高复现难度 baseline）

---

## 2. 评级阈值（按 `goal_level` 调档）

| goal_level | A 阈值 | B 阈值 | C 阈值 | D |
|---|---|---|---|---|
| 保毕业 | avg ≥ 70 | avg ≥ 55 | avg ≥ 40 | avg < 40 |
| 稳中求新 | avg ≥ 65 | avg ≥ 50 | avg ≥ 35 | avg < 35 |
| 冲高水平 | avg ≥ 60 | avg ≥ 45 | avg ≥ 30 | avg < 30 |

**逻辑**：保毕业要求最高（最严），冲高水平要求最低（最有冒险空间）。

---

## 3. 维度 1：方向成熟度（`papers + surveys + thesis_templates`）

| 项 | 加分（++）条件 | 加分（++）幅度 | 减分（--）条件 | 减分（--）说明 |
|---|---|---|---|---|
| 论文数 | ≥ 5 | +min(n×4, 40) | 1-2 篇 | 论文少 |
| 论文数 | 3-4 篇 | +min(n×4, 40) | — | — |
| 综述 | ≥ 1 | +min(n×10, 20) | 0 篇 | 缺综述 |
| 学位论文模板 | ≥ 1 | +min(n×15, 25) | — | — |
| 论文+综述齐 | 都 ≥ 1 | +15 | — | — |

**总分公式**：
```
s = min(papers×4, 40) + min(surveys×10, 20) + min(thesis_templates×15, 25) + (papers≥5 且 surveys≥1 时 +15)
s ∈ [0, 100]
```

---

## 4. 维度 2：数据可得性（`datasets + inherited_resources`）

| 项 | 加分（++）条件 | 加分（++）幅度 | 减分（--）条件 | 减分（--）说明 |
|---|---|---|---|---|
| 数据集候选 | ≥ 2 | +min(n×25, 60) | 1 个 | 数据集候选 < 2 偏少 |
| 数据集候选 | 1 个 | +min(n×25, 60) | 0 个 | 无数据集候选 |
| 继承资源可用 | ≥ 1 | +min(n×20, 40) | 标了不可用 | 继承资源未标可用 |
| 继承资源可用 | — | — | 0 项 | 无继承资源 (需自采数据) |

**总分公式**：
```
s = min(datasets×25, 60) + min(inherited_available×20, 40)
s ∈ [0, 100]
```

---

## 5. 维度 3：baseline 清晰度（`baselines × reproduce_difficulty`）

**复现难度打分**：
- 低：30 分/个
- 中：18 分/个
- 高：5 分/个
- 未知：12 分/个

**特殊情况**：0 个 baseline → s = 0

| 项 | 加分（++）条件 | 加分（++）幅度 | 减分（--）条件 | 减分（--）说明 |
|---|---|---|---|---|
| 复现难度=低 | ≥ 1 个 | +n_low × 30 | — | — |
| 复现难度=中 | ≥ 1 个 | +n_mid × 18 | — | — |
| baseline 总数 | ≥ 2 | +10 | < 2 | baseline 候选 < 2 |
| 复现难度=高 | — | — | ≥ 1 | n_high × 5 |
| 复现难度=未知 | — | — | ≥ 1 | 未知复现难度 |

**总分公式**：
```
diff_score = 低:30 + 中:18 + 高:5 + 未知:12  (按各 baseline 求和)
s = min(diff_score + 10, 100)  // +10 是 baseline 数 ≥ 1 的基础分
```

---

## 6. 维度 4：实验可行性（`metrics + experiment_templates`）

| 项 | 加分（++）条件 | 加分（++）幅度 | 减分（--）条件 | 减分（--）说明 |
|---|---|---|---|---|
| 评价指标 | ≥ 1 套 | +min(n×8, 50) | 0 | 无评价指标 (无法量化结果) |
| 实验模板 | ≥ 2 个 | +min(n×15, 40) | 1 个 | 实验模板 < 2 偏少 |
| 实验模板 | — | — | 0 | 无实验模板 |
| 指标可复现 | metrics 存在 | +10 | — | — |

**总分公式**：
```
s = min(metrics×8, 50) + min(exp_templates×15, 40) + (metrics 存在时 +10)
s ∈ [0, 100]
```

---

## 7. 维度 5：工作量可拆性（`work_package_drafts + wp_binding 覆盖`）

| 项 | 加分（++）条件 | 加分（++）幅度 | 减分（--）条件 | 减分（--）说明 |
|---|---|---|---|---|
| 工作包数 | ≥ 2 | +min(n×35, 70) | < 2 | 工作包数 < 2 |
| 证据已绑定 | ≥ 2 个 WP 被 evidence 引用 | +min(n×15, 30) | < 2 个被绑 | 证据未充分绑定 |

**绑定计算**：`evidence.wp_binding` 收集 papers/datasets/baselines 中所有引用的 WP，去重后计数。

**总分公式**：
```
s = min(work_packages×35, 70) + min(unique_bound_wps×15, 30)
s ∈ [0, 100]
```

---

## 8. 维度 6：毕业时间风险（`first_result_deadline + 高复现难度 baseline`）

**特殊情况**：无 `first_result_deadline` → s = 20，且报"未填首张结果表时间"

| 项 | 加分（++）条件 | 加分（++）幅度 | 减分（--）条件 | 减分（--）说明 |
|---|---|---|---|---|
| 高复现难度 baseline | 0 个 | +80（基线） | 1 个 | -15 拖进度 |
| 高复现难度 baseline | — | — | ≥ 2 个 | -(n×15) 时间红线危险 |
| 每周投入 | ≥ 25h | 信息项 | — | — |

**总分公式**：
```
s = 80 - (高复现难度 baseline × 15),  s ∈ [0, 100]
```

## 9. 总体流程

```
def build_risk_score(intake, spec, ledger) -> RiskScore:
    dims = [
        _score_maturity(ledger),
        _score_data(ledger, intake),
        _score_baseline(ledger),
        _score_experiment(ledger),
        _score_workload(spec, ledger),
        _score_time(ledger, intake),
    ]
    avg, rating = _overall(intake.goal_level, dims)  # 按 goal 选阈值
    min_dim = min(dims, key=lambda d: d.score)
    return RiskScore(
        ...,
        dimensions=dims,
        overall_score=avg,
        overall_rating=rating,
        max_risk_dimension=min_dim.key,  # 最弱维度 = 用户最该补的方向
        min_viable_path=_min_viable_path(intake, ledger, rating),
    )
```

`max_risk_dimension` = 6 维里分数最低的那一个。Phase 05 trace 面板会显示 "📊 六维风险评分"，最弱维度 + 总分 + 评级一并显示。

## 11. 当前已知问题（用户明确不深究，列出来供参考）

1. **评分函数是规则式**：没有考虑论文质量、影响力、引用数；只看"数量 + 字段是否填"
2. **wp_binding 计算粗**：只看 evidence.wp_binding 字段值，不分析 evidence 是否真支持 WP
3. **没有引入外部证据交叉验证**：不知道 evidence_ledger.papers 里的论文是否真存在
4. **6 维权重相同**：average 简单平均，没有 dimension-level 加权
5. **LLM 路径未启用**：现在 `prefer="heuristic"` 直接走规则，不调 M3 出 pivot；Pivot 是规则模板
