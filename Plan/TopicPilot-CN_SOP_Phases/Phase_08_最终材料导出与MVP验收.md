# Phase 08：最终材料导出与 MVP 验收

> 阶段目标：把前七个阶段的结构化产物整理为最终开题材料、答辩准备材料和项目演示验收包。  
> 方法依据：毕业论文合集强调“开题阶段要预埋最终论文目录、实验表格、参考文献分类和风险替代方案”。  
> 技术落点：`FinalReport`、导出服务、项目归档、MVP 验收清单。  
> 阶段输入：`ProposalDraft`、`CommitteeReview`、`WorkPackage`、`EvidenceLedger`。  
> 阶段产物：`07_final_package.md`、开题报告初稿、答辩问答清单、项目验收包。

---

## 1. 阶段定位

Phase 08 是收束阶段，不再继续扩张研究问题。

它要完成：

- 最终开题材料整理。
- 证据链与引用来源归档。
- 工作包、实验矩阵、风险预案归档。
- 开题答辩问答清单。
- MVP 演示验收标准。
- 后续毕业论文阶段路线图。

本阶段的目标不是把毕业论文写完，而是形成一套可开题、可演示、可继续推进到毕业论文的材料包。

---

## 2. 最终材料包内容

### 2.1 开题报告初稿

必须包含：

- 研究背景与意义
- 国内外研究现状
- 研究问题与目标
- 研究内容与技术路线
- 预期创新点
- 实验方案
- 可行性分析
- 进度安排
- 风险预案
- 参考文献框架

### 2.2 毕业论文骨架

五章式目录：

```text
第一章 绪论
第二章 相关基础
第三章 工作包一
第四章 工作包二
第五章 总结与展望
```

每章必须绑定：

- 内容来源
- 图表需求
- 实验需求
- 还缺什么材料

### 2.3 答辩问答清单

常见问题：

- 为什么选择这个题目？
- 当前研究现状是什么？
- 你的创新点是什么？
- 数据集和 baseline 从哪里来？
- 如果实验效果不好怎么办？
- 为什么这个工作量足够毕业？
- 你的系统和普通 LLM 生成有什么区别？

### 2.4 MVP 演示包

MVP 演示最小闭环：

```text
创建项目
→ Phase 01 评级
→ Phase 02 题目拆解
→ Phase 03 检索计划
→ Phase 04 证据账本
→ Phase 05 风险评分与 Pivot
→ Phase 06 工作包与实验矩阵
→ Phase 07 开题报告与审查
→ Phase 08 导出与归档
```

## 4. 当前后端验收标准

在没有 `apps/web` 前，Phase 08 的当前验收以材料和后端闭环为主：

- [ ] Phase 01-04 后端 smoke 已通过。
- [ ] Phase 05-07 产物均存在。
- [ ] 最终材料包包含开题报告、论文目录、工作包、创新点、实验矩阵、风险预案和问答清单。
- [ ] 所有关键材料能追溯到 EvidenceLedger、RiskScore 或 WorkPackage。
- [ ] 明确标记尚未完成的界面和 Playwright 验收项。

---

## 5. 界面与 Playwright 后置验收

当 `apps/web` 出现并可启动后，Phase 08 才把 Playwright 作为硬验收。

### 5.1 Happy Path

```text
创建 A/B 项目
→ 依次完成 Phase 01-08
→ 页面展示最终材料包
→ 刷新页面后材料仍可恢复
```

### 5.2 Blocked Path

```text
创建 C/D 项目
→ Phase 01 显示阻断或补问
→ 后续阶段按钮不可误触发
```

### 5.3 导出验收

- Markdown 可导出。
- 若支持 DOCX，DOCX 内容结构完整。
- 导出内容不包含未替换的 TBD。
- 导出材料中的证据来源可追溯。

---

## 6. 进入毕业论文执行阶段的条件

只有满足以下条件，才建议从“开题选题助手开发”进入“毕业论文执行”：

- 题目边界明确。
- 至少 1-2 个工作包定稿。
- 每个工作包都有实验矩阵。
- 数据、baseline、指标均已确定或有替代方案。
- 开题报告初稿已生成并通过审查。
- 风险预案明确。
- MVP 能演示核心闭环。

---

## 7. 与后续项目开发的关系

Phase 08 不是项目终点，而是开题阶段终点。

后续可以进入：

- 界面 MVP 开发与 Playwright 验收。
- DOCX / PDF 导出。
- 学校模板适配。
- 多案例评测数据集。
- Langfuse / OpenTelemetry 评估看板。
- 更完整的论文写作与毕业阶段工作流。
