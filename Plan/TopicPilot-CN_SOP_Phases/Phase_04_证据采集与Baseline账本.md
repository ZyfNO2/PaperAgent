# Phase 4：证据采集与 Baseline 账本

> 阶段目标：建立论文、数据、代码、指标、实验模板和可继承资源的证据账本。  
> 阶段输入：`02_query_plan.md`。  
> 阶段产物：`03_evidence_ledger.md`。

---

# 1. 阶段定位

毕业论文合集中的关键经验是：**先找基准，再谈方法；先确认实验入口，再谈创新点**。

因此 Phase 4 的证据采集不只是“搜到论文”，而要回答：

1. 有没有可用数据？
2. 有没有可运行 Baseline？
3. 有没有公认指标？
4. 有没有对比实验和消融实验模板？
5. 有没有同领域学位论文可参考章节结构？
6. 有没有课题组可继承资源？

---

# 2. 必采证据类型

| 类型 | 作用 |
|---|---|
| 论文证据 | 判断方向成熟度、研究问题和已有方法 |
| 综述证据 | 帮助写第一章研究现状层次 |
| 数据集证据 | 判断实验入口是否存在 |
| Baseline/代码证据 | 判断是否能复现或继承 |
| 指标证据 | 判断实验能否评价 |
| 实验模板证据 | 提取对比实验、消融实验、参数实验写法 |
| 学位论文模板证据 | 反推章节结构和工作量 |
| 资源证据 | 记录课题组/同门可继承资源 |

---

# 3. Baseline 账本要求

每个 Baseline 候选必须记录：

- 对应论文
- 代码仓库
- 是否有 README
- 是否有环境配置
- 是否有数据预处理脚本
- 是否有训练脚本
- 是否有评价脚本
- 是否有预训练权重
- 是否有人复现或维护
- 预计复现难度
- 是否适合学生当前资源

如果找不到可运行 Baseline，题目必须进入高风险状态，不能直接进入“创新点规划”。

# 5. 阶段验收

- 至少 10 篇相关论文，或明确证明精确方向论文不足。
- 至少 1 篇综述或研究现状类材料，或明确说明需要从多篇论文自建现状框架。
- 至少 2 个数据集候选，或明确证明数据不足。
- 至少 2 个 Baseline/代码候选，或明确证明代码不足。
- 至少 1 套评价指标。
- 至少 1 个对比实验或消融实验模板。
- 至少 1 篇同领域学位论文/目录模板，或明确说明暂无。
- 所有关键证据都有来源。
- 如果数据、Baseline、指标任一项为空，必须标记为后续硬风险。
