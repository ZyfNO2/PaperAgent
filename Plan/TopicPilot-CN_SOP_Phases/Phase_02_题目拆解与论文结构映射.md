# Phase 2：题目拆解与论文结构映射

> 阶段目标：把题目拆成研究组件，并预判它是否能形成毕业论文工作量。  
> 阶段输入：`00_input.md`。  
> 阶段产物：`01_topic_spec.md`。

---

# 1. 阶段定位

毕业论文合集中的重要经验是：**大论文通常不是一个抽象想法，而是若干可写章节的组合**。

因此 Phase 2 不能只做技术拆解，还要同步判断：

- 这个题目能不能拆出 2～3 个工作包？
- 能不能对应第三章、第四章和系统/泛化章节？
- 是不是只有一个很大的口号，没有实验入口？
- 是否只是“换模块”，但没有真实研究问题？

---

# 2. 执行步骤

## Step 1：拆研究组件

提取以下字段：

| 字段 | 说明 |
|---|---|
| 研究对象 | 研究针对的实体或场景 |
| 应用场景 | 工程/业务/学术背景 |
| 核心任务 | 检测、分类、分割、生成、预测、推荐等 |
| 数据模态 | 文本、图像、视频、音频、点云、多模态等 |
| 方法族 | 传统方法、深度学习、Transformer、LLM、RAG、Agent 等 |
| 预期输出 | 标签、掩膜、分数、报告、系统结果等 |
| 评价指标 | Accuracy、F1、IoU、BLEU、延迟、人评等 |
| 工程约束 | 算力、实时性、设备、部署、数据权限 |

## Step 2：识别高风险词

重点标记：

```text
智能、高精度、实时、全场景、多模态、自适应、端到端、通用、大模型、数字孪生、精确测量
```

每个高风险词都必须转成可验证定义。

## Step 3：做论文结构预映射

将题目初步映射到论文结构：

```text
第一章：背景、意义、研究现状
第二章：相关理论，只写真正会用到的基础
第三章：数据/基线/问题分析或方法一
第四章：方法改进、对比实验、消融实验或方法二
第五章：系统、泛化、总结与展望
```

## Step 4：判断工作包雏形

参考毕业论文合集中的“两篇小论文/两个工作量”思路，但转化为合规表达：

```text
工作包 = 一个明确问题 + 一个方法/实验闭环 + 一个章节位置
```

# 4. 阶段验收

- 至少拆出对象、任务、数据、方法、评价五项。
- 每个高风险词都有可验证定义或待追问问题。
- 至少提出 2 个工作包雏形，或明确说明为什么拆不出。
- 已完成论文结构预映射。
- 若第三章/第四章没有可写内容，必须标为高风险。
- 不得把题目直接美化成结论。
