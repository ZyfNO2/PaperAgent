# Phase 1：任务建档与毕业目标确认

> 阶段目标：先定毕业目标，再定题目风险上限。  
> 阶段输入：学生原始想法、导师要求、毕业时间、资源条件。  
> 阶段产物：`00_input.md`。  
> 阶段结论：`A 输入完整 / B 基本可用 / C 关键缺失 / D 当前不可评估`。

---

# 1. 阶段定位

毕业论文合集中的一个核心经验是：**方向选择不是单纯的学术问题，而是目标管理问题**。

因此 TopicPilot-CN 的第一阶段不急着拆题，也不急着搜索论文，而是先回答：

1. 学生到底要保毕业、稳中求新，还是冲高水平？
2. 导师和学院对题目有什么硬性要求？
3. 当前数据、代码、算力、时间是否支持这个目标？
4. 题目最终能不能形成 2～3 个可写入论文章节的工作量？

如果 Phase 1 做不清楚，后续很容易出现“题目看起来很漂亮，但没有数据、没有 Baseline、没有实验、没有章节”的问题。

# 2. 必须收集的信息

## 2.1 毕业目标与风险档位

```markdown
- 目标档位：保毕业 / 稳中求新 / 冲高水平
- 是否要求小论文：
- 是否要求系统、专利、软著或工程实践：

- 可接受风险：低 / 中 / 高
- 最晚什么时候必须得到第一张实验结果表：
```

目标档位解释：

| 目标档位 | 选题策略 | 不建议做 |
|---|---|---|
| 保毕业 | 选成熟方向，有论文、有数据、有代码、有指标 | 原始创新、从零采数据、从零复现大模型 |
| 稳中求新 | 成熟任务 + 局部改进 + 场景迁移 | 多个高风险点同时推进 |
| 冲高水平 | 可以接受探索风险，但必须保留最小可行实验 | 没有退路的全串行题目 |

# 4. 执行步骤

## Step 1：建立案例编号

格式：

```text
YYYYMMDD_专业简称_题目关键词
```

示例：

```text
20260615_CV_桥梁裂缝检测
```

## Step 2：先定目标档位

必须先问清楚目标档位。毕业论文合集的经验可以转化为以下规则：

- 保毕业：优先找“人多方向”，即论文多、代码多、数据多、可模仿模板多。
- 稳中求新：在成熟方向里找一个明确问题做局部改进。
- 冲高水平：可以探索新人少方向，但必须保留一个保底工作包。

## Step 3：确认可继承资源

重点确认：

- 有没有同门论文目录可以参考
- 有没有师兄师姐代码可以合法复用
- 有没有已有数据或项目数据
- 有没有已经跑通的环境

如果没有可继承资源，后续 Phase 3 和 Phase 4 必须把“找可运行 Baseline”作为第一优先级。

## Step 4：确认时间红线

必须记录：

- 距离开题还有多久
- 距离毕业还有多久
- 最晚什么时候要有第一张实验表

如果第一张结果表的时间无法安排，题目不能进入高风险路线。

## Step 5：记录待补问题

所有不确定项都要写进待补问题，不能在后续阶段悄悄假设。

# 5. Phase 1 验收标准

## 5.1 文件验收

`00_input.md` 必须存在，并包含以下 8 个部分：

- 案例基本信息
- 毕业目标与风险档位
- 导师与课题组条件
- 学生资源
- 原始题目与取舍
- 初步毕业可行性直觉
- 待补问题
- Phase 1 结论

## 5.2 核心字段验收

| 字段 | 验收要求 |
|---|---|
| 原始题目 | 至少有一个完整题目或方向描述 |
| 目标档位 | 明确为保毕业、稳中求新或冲高水平 |
| 开题时间 | 至少给出月份或相对时间 |
| 预计毕业时间 | 至少给出月份或相对时间 |
| 导师方向 | 至少给出一个方向关键词 |
| 数据条件 | 明确有、无或不确定 |
| 代码条件 | 明确有、无或不确定 |
| 算力条件 | 明确 GPU、CPU、云资源或不确定 |
| 可继承资源 | 明确有无同门论文、代码或数据 |
| 第一张结果表时间 | 明确最晚时间或列为待补 |

## 5.3 输入完整性评级

```text
A：输入完整
可直接进入 Phase 2。

B：基本可用
可以进入 Phase 2，但必须带着明确假设。

C：关键缺失
不建议进入 Phase 2，必须先补问。

D：当前不可评估
题目、目标、毕业时间、导师方向或资源条件严重缺失。
```

## 5.4 允许进入 Phase 2 的条件

只有以下情况允许进入 Phase 2：

- 评级为 A
- 评级为 B，且所有缺失项都已写成显式假设

## 5.5 阻断条件

出现以下情况时，Phase 1 必须阻断：

1. 只有一句“帮我想个题目”，没有专业、导师方向或毕业时间。
2. 用户不提供资源条件，却要求判断能否毕业。
3. 已临近毕业，但题目要求从零采集、标注、训练和部署。
4. 目标档位是保毕业，但用户坚持多个高风险创新点同时推进。
5. 用户要求编造论文、数据或实验结果。
