# Phase 3：方向成熟度与检索计划

> 阶段目标：生成检索计划，并判断方向是否属于“人多、基准多、模板多”的成熟区域。  
> 阶段输入：`01_topic_spec.md`。  
> 阶段产物：`02_query_plan.md`。

---

# 1. 阶段定位

毕业论文合集中的方向选择经验可以转化为一句话：

> 保毕业优先选择成熟方向；成熟方向的标志是论文多、数据多、代码多、实验套路多、同领域论文模板多。

因此 Phase 3 不只是列关键词，而是要为 Phase 4 的证据采集设计检索路线，尤其要找：

1. 近 3～5 年论文密度
2. 综述和经典论文
3. 数据集和 Benchmark
4. 可运行 Baseline
5. 同领域硕博论文目录模板
6. 对比实验和消融实验写法

---

# 2. 检索层级

```text
L0 原始精确题目
L1 中英文同义词
L2 去掉特殊场景后的通用任务
L3 底层方法或任务族
L4 数据集 / Baseline / Benchmark 专项检索
L5 学位论文 / 章节结构 / 实验模板检索
```

---

# 3. 检索策略

## 3.1 方向成熟度检索

用于判断“做的人多不多”：

```text
核心英文任务词 + survey
核心英文任务词 + review
核心英文任务词 + benchmark
核心英文任务词 + dataset
核心英文任务词 + GitHub
核心英文任务词 + baseline
```

## 3.2 Baseline 优先检索

毕业论文合集强调先找基准模型。这里转化为：

```text
任务词 + "code"
任务词 + "implementation"
任务词 + "github"
论文标题 + "github"
数据集名 + "baseline"
方法名 + "pretrained"
```

## 3.3 论文结构模板检索

用于后续章节映射：

```text
中文方向词 + 硕士论文
中文方向词 + 学位论文
英文方向词 + thesis
英文方向词 + dissertation
方向词 + 消融实验
方向词 + 对比实验
```

# 5. 阶段验收

- 至少 10 个英文检索词组合。
- 至少 5 个中文检索词组合。
- 必须覆盖论文、综述、数据集、Baseline、Benchmark。
- 必须包含“学位论文/目录/实验模板”检索。
- 每个工作包雏形至少有一组检索词。
- 如果无法生成 Baseline 检索词，必须标记为高风险。
