# Phase 06：工作包定稿与实验矩阵

> 阶段目标：把已通过风险决策的题目转化为 1-2 个可写入毕业论文第三章、第四章的工作包，并为每个工作包绑定实验矩阵。  
> 方法依据：毕业论文合集强调“大论文通常由两个工作量组成”“创新点必须绑定实验”“每个章节绑定资料来源”。  
> 技术落点：`WorkPackage`、`ExperimentMatrix`、`WorkPackageGraph`。  
> 阶段输入：`EvidenceLedger`、`RiskScore`、`PivotCandidate` 或继续当前题目的决策。  
> 阶段产物：`05_work_packages.md`、正式工作包、实验矩阵、论文目录映射。

---

## 1. 阶段定位

Phase 06 是从“题目能不能做”进入“具体做什么”的阶段。

它要把 Phase 05 的决策落成：

- 第三章写什么。
- 第四章写什么。
- 每个工作包解决什么问题。
- 每个工作包用什么方法。
- 每个工作包依赖哪些论文、数据、baseline、指标。
- 每个工作包需要哪些对比实验、消融实验、参数实验和案例分析。

毕业论文合集里最关键的经验是：

> 工作量不是一句创新点，而是一个问题 + 一个方法 + 一个实验闭环 + 一个章节位置。

---

## 2. 工作包类型

可选工作包类型：

| 类型 | 适用场景 | 典型实验 |
|---|---|---|
| 证据链构建型 | 题目推荐缺少可追溯依据 | 证据准确率、覆盖率、人工评审 |
| 风险评分型 | 学生难判断题目是否能毕业 | 风险分类、案例一致性、误判分析 |
| Pivot 决策型 | 题目过大或证据不足 | Pivot 接受率、前后风险对比 |
| 系统实现型 | 需要展示完整工程能力 | 端到端流程、延迟、可用性 |
| 对比分析型 | 方法创新有限但证据充分 | 多方法对比、误差分析 |
| 模板生成型 | 开题报告材料生成 | 章节完整度、引用 grounding、人工评分 |

---

## 3. 实验矩阵要求

每个工作包至少绑定：

- 1 个研究问题
- 1 个方法方案
- 1 组数据或案例
- 1 个 baseline 或对照方案
- 1 套评价指标
- 1 个主实验
- 1 个消融、参数或案例分析
- 1 个论文章节位置

创新点必须绑定实验：

```text
模块替换 → 消融实验
性能提升 → 对比实验
稳定性提升 → 多案例 / 多数据集实验
解释性提升 → 案例分析 / 可视化
系统能力 → 端到端流程与可用性测试
```

## 4. 推荐 TopicPilot-CN 工作包

### WP1：多证据链选题可行性评估

研究问题：

> 传统开题选题建议往往只给题目或泛泛理由，缺少文献、数据集、baseline 和指标支撑。

方法方案：

- 基于 Phase 02-04 的结构化产物构建证据链。
- 将论文、数据集、baseline、指标、学位论文模板绑定到题目和工作包。
- 输出可解释的选题可行性判断。

实验矩阵：

- 主实验：证据链完整度评估。
- 对比实验：无证据推荐 vs 证据链推荐。
- 案例分析：A/B/C/D 不同风险案例。
- 指标：Evidence Precision、Baseline Recall@K、Citation Hallucination Rate。

章节位置：第三章。

### WP2：面向毕业风险的 Pivot 与工作包生成

研究问题：

> 学生即使知道题目有风险，也难以把题目收缩成可毕业的工作包。

方法方案：

- 构建六维风险评分。
- 针对 C/D 风险生成收缩型或换向型 Pivot。
- 将 Pivot 后题目映射为第三章、第四章工作包。

实验矩阵：

- 主实验：风险评分与人工判断一致性。
- 对比实验：原题 vs Pivot 后题目风险变化。
- 消融实验：去掉 baseline / 数据 / 时间红线维度后的评分变化。
- 案例分析：题目过大、无 baseline、无数据三类失败案例。

章节位置：第四章。

## 6. 当前验收标准

- [ ] 至少形成 1 个完整工作包，目标是 2 个。
- [ ] 每个工作包都有研究问题、方法方案、数据、baseline、指标和章节位置。
- [ ] 每个创新点绑定至少 1 个实验。
- [ ] 第三章、第四章均有可写内容；若只有一个工作包，必须说明补足策略。
- [ ] 实验矩阵包含主实验和至少一种补充实验。
- [ ] 五章式论文目录映射完整。
- [ ] 不允许把“系统实现”当作唯一工作量，除非有清晰评价指标。

## 7. 界面 MVP 后置验收

当前无浏览器页面，Playwright 不作为即时验收条件。

界面出现后，应补充：

- 工作包卡片可见。
- 实验矩阵表可见。
- 用户能选择保留 / 删除 / 调整工作包。
- 章节映射变化能实时反映。
- 刷新后工作包定稿状态可恢复。
