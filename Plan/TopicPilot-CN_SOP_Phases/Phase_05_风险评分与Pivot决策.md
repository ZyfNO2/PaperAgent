# Phase 05：风险评分与 Pivot 决策

> 阶段目标：把 Phase 04 的证据账本转化为可解释的选题风险评分，并给出继续、收缩或转向的决策建议。  
> 方法依据：毕业论文合集强调“不要造航母”“题目过大要收缩到具体任务”“先确认数据、baseline、指标，再谈创新”。  
> 技术落点：`RiskScore`、`PivotCandidate`、`RiskEvaluationGraph`、`PivotPlanningGraph`。  
> 阶段输入：Phase 04 验收通过的 `EvidenceLedger`。  
> 阶段产物：`04_risk_and_pivot.md`、结构化风险评分、Pivot 候选方案。

---

## 1. 阶段定位

Phase 05 是“能不能继续做这个题目”的决策阶段。

Phase 01-04 已经完成：

- 学生目标与资源建档
- 题目拆解与论文结构映射
- 检索计划
- 论文、数据集、baseline、指标与模板证据账本

Phase 05 不再继续堆证据，而是回答：

- 当前题目是否能稳定毕业？
- 是否存在“航母题目”风险？
- 数据、baseline、指标、时间红线是否支撑后续实验？
- 是否需要 Pivot？
- 如果 Pivot，应当收缩到哪个更可毕业的方向？

---

## 2. 毕业方法论转化

| 毕业论文合集经验 | Phase 05 中的产品化规则 |
|---|---|
| 稳定毕业优先成熟方向 | 成熟度、证据可得性、baseline 可复现性进入评分 |
| 题目过大要收缩 | 输出 PivotCandidate，不只给风险提示 |
| 文献多但不可复现仍高风险 | baseline 与数据权重高于论文数量 |
| 创新点必须能验证 | 无法绑定实验的创新不进入后续工作包 |
| 第一张结果表是生死线 | baseline 复现周期必须对照 Phase 01 时间红线 |

---

## 3. 风险评分维度

建议六维评分，每项 0-100 分：

| 维度 | 含义 | 主要证据 |
|---|---|---|
| 方向成熟度 | 近年论文、综述、模板是否充分 | papers、surveys、thesis_templates |
| 数据可得性 | 是否有公开数据或可获得数据 | datasets、继承资源 |
| baseline 清晰度 | 是否有可运行代码和可比较方法 | baselines |
| 实验可行性 | 指标、对比、消融能否成立 | metrics、experiment_templates |
| 工作量可拆性 | 是否能形成第三章、第四章工作包 | work_package_drafts、wp_binding |
| 毕业时间风险 | 是否能在时间红线前得到结果表 | first_result_deadline、reproduce_difficulty |

综合风险等级：

```text
A：低风险，可继续当前题目
B：中低风险，可继续，但需保留补救方案
C：中高风险，必须收缩或补证据
D：高风险，不建议继续当前题目
```

---

## 4. Pivot 决策规则

### 4.1 不需要 Pivot

满足以下条件时，可继续当前题目：

- 数据集不少于 1 个可用候选。
- baseline 不少于 1 个可复现候选。
- 至少 1 套评价指标。
- 工作包能映射到第三章和第四章。
- baseline 复现周期不超过时间红线。

### 4.2 需要收缩型 Pivot

适用于题目过大但证据仍可用：

```text
通用场景 → 具体场景
全自动系统 → 人工确认辅助系统
大模型训练 → RAG / Agent 工作流
多任务 → 单一核心任务
全量数据 → 公开数据集 / demo 数据集
```

### 4.3 需要换向型 Pivot

适用于关键证据缺失：

- 没有数据，也没有替代数据。
- 没有 baseline，也没有可比较规则方法。
- 没有评价指标。
- 题目与导师方向严重偏离。
- 时间红线无法支撑当前实验。

- [ ] 不允许编造论文、数据、baseline 或实验结果。

## 7. 界面 MVP 后置验收

当前无 `apps/web`，因此 Playwright 不作为本阶段即时验收条件。

界面出现后，应补充：

- 风险雷达图或六维评分表可见。
- 用户能看到每个评分的证据依据。
- 用户能在继续、收缩、转向候选中选择一个路线。
- C/D 风险时，继续按钮需要二次确认。
- 页面刷新后风险评分与 Pivot 选择可恢复。
