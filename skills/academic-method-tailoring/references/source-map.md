## 2. Retained ideas

| Source idea | Skill translation |
|---|---|
| A+B+C modular view | Baseline plus explicitly attributed interventions |
| Find a runnable baseline | Reproduction gate with commit, environment, split, and metric |
| Read methods by overview, modules, and equations | Data-flow-first method audit with symbol and interface contracts |
| Track input/output shape | Full semantic contract: shape, dtype, scale, masks, order, and gradients |
| Add modules one at a time | Feature-gated incremental implementation and ablation |
| Problem plus solution defines innovation | Falsifiable Problem–Method–Insight claim |
| Comparison, ablation, and case study | Fair experiment matrix plus mechanism and failure-case analysis |
| Write the body before abstract packaging | Write claims only after evidence and experiment gates |

## 3. Rejected or repaired ideas

| Unsafe or weak source advice | Required repair |
|---|---|
| Copy generic method prose | Learn structure, then write original prose with citations |
| Choose only weak comparisons or ignore stronger known work | Use a documented search protocol and relevant strong comparisons |
| Hide limitations or “soft spots” | Report limitations, negative results, and failure conditions |
| Treat shape alignment as sufficient | Verify semantics, ordering, normalization, masks, optimization, and gradients |
| Call any module combination innovation | Require a sourced gap, mechanism, prediction, and falsifier |
| Use unofficial or unauthorized download channels | Use legal institutional, author, preprint, or open-access routes |
| Inherit code without ownership clarity | Verify license, permission, attribution, and contribution history |
| Present unrun experiments as objective description | Record only executed runs and preserve provenance |
| Define SOTA as “what you know” | Search systematically and disclose scope and cutoff date |

## 4. Derived workflow

The source collection gives a useful decomposition but not a sufficient research method. This skill adds the missing controls:

1. evidence ledger;
2. reproducibility freeze;
3. semantic interface contracts;
4. falsifiable novelty claims;
5. fair and discriminating experiments;
6. claim–evidence traceability;
7. explicit go/revise/no-go decisions.

These controls convert “academic tailoring” from surface-level assembly into auditable incremental research.
