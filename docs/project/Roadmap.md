# Roadmap

> v0.1.0-rc1 → v1.0 路线图. 每个版本明确边界与"不做什么".

## v0.1 — 开题证据工作台 MVP ✅ 当前

**完成态**: 19 个 Session, 1 RC. Evidence workbench + FinalPackage + 报告模板.

**核心能力**:
- 题目理解 → 关键词拆解 → 三线检索 → 证据评分 → 双栏工作台
- EvidenceRef 追溯, Markdown 开题报告 (13 章节)
- 3 轻量学校模板 (default / engineering / cv_ai)
- 错误处理 / 空状态 / 可观测性
- Demo baseline + 回归基线

**边界**:
- 仍不自动代写论文正文
- 仍不伪造引用
- 仍保留用户确认 (Human Gate)
- 仍不做 DOCX 精排
- 仍不做全文向量检索

---

## v0.2 — 可选学校模板 / DOCX 导出

**目标**: 适配更多学校模板; 输出 Word 文档.

**新增能力**:
- DOCX 导出 (基于 `python-docx`)
- 学校模板扩展: 5-10 个常见模板
- 模板字段映射 (用户上传学校模板 → 自动识别字段)

**边界**:
- DOCX 仅做结构化导出, 不做 Word VBA / 宏
- 模板仍为静态骨架, 不做可视化编辑
- LLM 仍只做辅助, 不代写正文

---

## v0.3 — 轻量全文片段检索

**目标**: 引入 RAG, 让证据不只是元数据, 还有全文片段.

**新增能力**:
- 全文 embedding 索引 (Chroma / LanceDB)
- 章节级 chunk 检索
- 证据回链到 PDF / 网页具体段落

**边界**:
- 不引入重型向量库 (Milvus / Pinecone)
- 不做 cross-encoder rerank
- 不做多模态 embedding

---

## v0.4 — 更强资料解析

**目标**: 让 PDF / 图片 / 网页 真正"可读".

**新增能力**:
- OCR (PaddleOCR / Tesseract)
- 网页正文抽取 (trafilatura)
- 表格 / 图片识别
- 资料自动分类 (论文 / 博客 / 官方文档)

**边界**:
- 不做视频 / 音频解析
- 不做版式还原 (LaTeX / Word 样式)
- 不做翻译

---

## v1.0 — 稳定多项目管理与部署

**目标**: 可对外发布的稳定版本.

**新增能力**:
- 多项目隔离 (账号 / 工作空间)
- 协作 (评论 / 共享)
- Docker 镜像 + CI/CD
- 监控 / 告警 / 限流
- 公网部署 (HTTPS / WAF)

**边界**:
- 不做移动端 App
- 不做企业级权限 (RBAC)
- 不做计费 / 商业化

---

## 通用不变式 (跨所有版本)

1. **不自动代写论文** — 仅辅助开题报告 + 证据收集.
2. **不伪造引用** — 所有引用必须有 evidence_id + url_verified.
3. **保留用户确认** — Human Gate 在关键节点不取消.
4. **LLM 可降级** — heuristic fallback 永远在.
5. **隐私优先** — 不向第三方上传用户文件, 凭据不入 git.
