## 模板 A：项目导向（推荐）

> 我做了一个叫 PaperAgent 的开题证据 Agent，主要解决**「选这个论文题能不能开」**的问题。
>
> 流程是：输入一个题目 → 自动拆关键词 → 论文/数据集/仓库三线检索 → 7 维风险评估 → 输出可导出的开题报告。
>
> 技术上做了三件比较有意思的事：
> 1. **证据治理闸门** — 候选 ≠ 证据，URL 必须验证才能晋升
> 2. **4 层 Agent Memory** — 短/中/长/不可变，关键事件压缩不丢
> 3. **8 维 Readiness 硬拦截** — 风险章节、夸大意词、缺数据集直接阻断导出
>
> 项目覆盖 ~470+ 后端测试 + ~30 个 Playwright E2E，能稳定跑完整链路。
> 当前是 MVP 阶段，主要瓶颈是真实 embedding / 向量库未接入。

**字数：** 约 240 字（CJK）

## 模板 B：技术导向

> PaperAgent 是我独立设计的科研证据 Agent，基于 FastAPI + Vue 3 全栈。
>
> 核心是**多阶段流式 Workflow**：8 个 Step Deck 串成 pipeline，每个 Step 都有 Gate 校验。技术上实现了 4 层 Agent Memory（ShortContext / Transcript / ProjectMemory / EvidenceMemory），关键事件压缩 100% 保留。
>
> 最近在做的是**面试级 RAG**：Hybrid 检索 + 5 因子 Rerank + 8 评估指标 + 5 个失败检测器，评估不是花架子而是真的能定位问题。
>
> 同时把核心能力暴露为 **4 个 MCP tools**，3 层权限（白名单 + 黑名单 + Gate）保证外部 Agent 不能调高风险动作。
>
> 这个项目对我最大的训练是「**工程边界感**」——知道什么让 AI 做、什么不让 AI 做、什么是 hard block、什么是降级。

**字数：** 约 270 字（CJK）

## 模板 C：业务导向

> 我做的是给本科生 / 研究生用的开题辅助 Agent。
>
> 核心痛点是：**同学选题拍脑袋，开题答辩被老师挑出问题**。PaperAgent 把 7 维风险评估、URL 验证、夸大意词检测做进系统，**强制要求报告满足 8 维 Readiness 才能导出**。
>
> 这不是又一个 RAG 玩具，是真的能拦住不合格报告的工具。我的工程实现保证了它的稳定性：~470 测试覆盖 8 个 step，LLM 挂了有 heuristic fallback，刷新页面能自动 replay 恢复。
>
> 如果继续做，优先补真实 embedding 和向量库。
