# PaperAgent · 3分钟自我介绍（中文）

> 适用于：技术面试开场 / 项目深挖铺垫
> 时长：3 分钟（约 600-800 字 CJK）

## 完整版（推荐）

### 开场（30 秒）— 项目是什么 + 为什么

> 我做的是 PaperAgent——一个**科研开题证据 Agent**，主要解决本科生和研究生在选论文题时「**拍脑袋选 → 答辩被挑出问题**」的痛点。
>
> 核心思路：把开题这件事从「人肉判断」变成「**可追溯、可审计、可降级**」的工程闭环。系统会强制检查 7 维风险（数据集、Baseline、指标等），不满足就不让导报告。

### 项目背景（30 秒）— 现状问题

> 现在市面上的工具，要么是 LLM 聊天框——容易编论文；要么是搜索引擎——返回一堆候选没法用。PaperAgent 想做的事情是**用工程化的方式把"开题决策"做严谨**：
>
> - 候选资源**必须 URL 可访问**才能晋升
> - 报告**必须 8 维 readiness 全绿**才能导出
> - 任何 LLM 决策都有**Heuristic 兜底**，LLM 挂了不中断

### 技术架构（60 秒）— 怎么做的

> 技术上我用 FastAPI + Vue 3 全栈，后端 ~10 个 Pydantic schema 定义数据契约，~30 个 service 模块实现业务逻辑，~470 个 pytest 测试 + ~30 个 Playwright E2E 覆盖。
>
> 整体是一个 8 步 Step Deck：题目输入 → 关键词拆解 → 检索词生成 → 三线检索 → 候选评分 → 可行性裁决 → 证据晋升 → 报告草稿。每步都有 Gate 校验，前一步没过下一步 409 拒。
>
> 几个比较有意思的设计：
> - **4 层 Agent Memory**：短/中/长/不可变，刷新页面能自动 replay 恢复
> - **面试级 RAG**：Hybrid 检索 + 5 因子 Rerank + 8 评估指标 + 5 个失败检测器
> - **MCP Server 暴露 4 个最小 tools**：3 层权限（白名单 + 黑名单 + Gate），外部 Agent 调高风险动作直接拒

### 难点 + 应对（45 秒）

> 项目里最难的不是单点技术，是**「工程边界感」**——什么让 LLM 做、什么不让 LLM 做、什么是 hard block、什么是降级。
>
> 比如：
> - 夸大意词检测——「填补空白」这种词直接阻断导出
> - 风险章节缺失——直接阻断导出
> - 检索为空——不假装找到了，明确告诉用户
> - Multi-Agent 成本超限——自动降级到单流程
>
> 这些边界不是事后加的，是从设计上就内化进系统的。

### 测试与可靠性（30 秒）

> 我对项目可靠性的投入可能比功能还多：
> - ~470 后端测试，每个 session commit 前必跑
> - ~30 个 Playwright E2E，模拟真实用户操作
> - 真实 uvicorn 启动 + smoke 验证
> - S31 baseline 专门测，确保不回退
>
> 4 层 Memory 的 critical 事件 100% 保留测试也覆盖了。

### 收尾（15 秒）

> 当前是 MVP 阶段，主要瓶颈是真实 embedding / 向量库未接入。如果继续做，优先补 RAG 的真 embedding、Memory 持久化到 SQLite、MCP stdio transport。
>
> 欢迎深挖任何模块。

## 简化版（2 分钟）

> 我做的是 PaperAgent——**科研开题证据 Agent**，把开题决策从「人肉判断」变成「**可追溯、可审计、可降级**」的工程闭环。
>
> 流程是 8 步：题目 → 关键词 → 检索 → 候选 → 可行性 → 证据晋升 → 报告草稿。每步有 Gate，前一步不通过下步 409 拒。
>
> 技术上几个亮点：
> - **证据治理** — 候选 ≠ 证据，URL 必须验证才能晋升
> - **4 层 Agent Memory** — 关键事件压缩不丢，刷新自动 replay
> - **8 维 Readiness** — 夸大意词 / 风险章节缺失 / 缺数据集都阻断导出
> - **MCP 暴露 4 个 tools** — 3 层权限防止越权
> - **面试级 RAG** — Hybrid 检索 + 8 评估指标 + 5 失败检测器
>
> 投入最多的是测试和工程边界：~470 后端测试 + ~30 Playwright E2E，每个 session commit 前必跑。
>
> 如果继续做，优先补真实 embedding + Memory 持久化。
